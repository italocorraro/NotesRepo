Appunti sull'elemento head in html

Questa pagina copre il contenuto dell'elemento <head> in html.

Overview

L'elemento head è una parte del documento che non viene mostrata dal browser, ma le informazioni al suo interno vengono utilizzate dal browser per visualizzare correttamente il documento.

<title>

Questo elemento racchiude il titolo visualizzato dal browser:

<base>

Questo elemento serve ad indicare un URL base e il target per tutti gli URL relativi nel documento.

Vedere più in dettaglio qui.

<style>

L'elemento style si può utilizzare per creare una stylesheet in CSS localmente.


questo elemento non può importare fogli di stile esterni; per farlo si usa link.

<script>

Questo elemento è usato per creare script all'interno del documento.


Importare script esterni

Attraverso l'attributo src è possibile importare fogli di script esterni:

alcuni script necessitano dell'attributo defer per essere eseguiti correttamente; defer è un attributo booleano che specifica che lo script va scaricato in parallelo con il parsing del documento, MA va eseguito solo al termine del parsing.
Questa è una necessità spesso dovuta alla mancanza di alcune cose necessarie all'esecuzione dello script prima del parsing.

<link>

Questo elemento serve a collegare il documento con uno esterno:

  • l'attributo rel indica il tipo di relazione tra il documento e quello collegato.
  • l'attributo href specifica la posizione, tramite URL, del documento collegato.
  • l'attributo media specifica su quale dispositivo il documento collegato verrà utilizzato.
  • l'attributo sizes indica le dimensioni della risorsa collegata

    questo attributo funziona solo se rel="icon"

  • l'attributo type indica il tipo di media del documento collegato.


Qui è usato per per indicare al browser la versione principale dell'URL della pagina nel caso ce ne fossero altri a chiamare lo stesso documento html.


Qui è usato per indicare il json che contiene informazioni per il il browser su come rendere una Progressive Web App.

Importare fogli di stile CSS

L'elemento link permette di importare un file esterno in formato .css da applicare alla pagina.


questo elemento non può importare qualsiasi tipo di file, in particolare non importa i file JavaScript.

<meta>

I metadati sono informazioni atte a descrivere i dati di un documento; il tag meta è il metodo ufficiale per aggiungere questo tipo di dati; sono svariati i tipi di informazione che porta, i più importanti sono:

  • charset serve a specificare il set di caratteri da utilizzare nel documento;

    UTF-8 è quello più usato in html poiché comprende caratteri da praticamente tutte le lingue.

  • La viewport è la parte visibile della pagina; indicare queste impostazioni esattamente come riportato permette al browser di rendere la pagina nelle giuste dimensioni anche da schermi più piccoli.

    mancare di immettere questa informazione comporta una visualizzazione incorretta su browser mobile.

  • description è utilizzata dai motori di ricerca; è quella breve descrizione sotto il titolo della pagina che viene mostrata dopo una ricerca
  • author è utilizzata per indicare l'autore della pagina.
  • Questo serve a indicare compatibilità con Internet Explorer.
  • keywords serve a indicare delle parole chiave per i motori di ricerca.

    ormai non è più tenuto in considerazione dai motori di ricerca moderni per via della facilità con cui poteva essere abusato.

è quasi un requisito in una pagina web impostare almeno il charset e il viewport.

Open Graph Data

Questo è un protocollo di metadati creato da Meta per mostrare metadati quando vengono riportato l'URL della pagina su un software che lo supporta:

  • og:title riporta il titolo.
  • og:type riporta il tipo dell'oggetto.
  • og:image riporta un'immagine nel grafo.
  • og:url riporta l'url canonico dell'oggetto.


qui sono riportati altri metadati supportati dal protocollo.
