Appunti sulla sintassi del CSS

Questa pagina copre l'argomento della sintassi base del CSS.

Overview

Il CSS (Cascading Style-Sheets) è un linguaggio utilizzato per controllare l'aspetto del contenuto visualizzato a schermo da un Browser;
CSS permette di formattare il testo, creare layout e anche di creare alcune animazioni e transizioni.

Sintassi Base

Il CSS non è un vero linguaggio di programmazione, ma piuttosto un documento che raccoglie una serie di regole che il browser deve seguire per visualizzare i contenuti.

Regole di Stile

Le regole di CSS hanno una struttura semplice:

selector { property-1: property-1-value; property-2: property-2-value; }

Il selettore seleziona uno o più elementi html a cui applicare gli stili nelle parentesi graffe; ogni stile è identificato da una proprietà a cui si associa un valore.

Le varie proprietà CSS permettono diversi valori.

Commenti

I commenti in CSS devono essere chiusi tra /* e */


@rules

Le at-rules sevono per dare istruzioni sul comportamento del CSS.

Shorthand

Alcune proprietà possono impostare più stili contemporaneamente; sono dette shorthand e fondamentalmente riassumono delle sotto-proprietà analoghe in una scrittura più compatta.

Organizzazione

Dato che la scrittura del CSS è estremamente monotona, è bene dividere il foglio di stile in sotto-sezioni logiche, usando dei commenti per dichiarare la logica (come un titolo), in modo da poter più facilmente navigare il foglio in caso fosse necessario modificarlo o espanderlo

/* Foglio di stile per le pagine <pagine> */

/* || Reset Globali */

Qui inserire i reset per i default indesiderati...

/* || Variabili Globali */

Qui inserire le variabili di uso comune nel file...

/* || Impostazioni Globali */

Qui inserire gli stili base per body, p...

/* || Stili Header */

Qui inserire gli stili per l'header...

/* || Stili Footer */

Qui inserire gli stili per il footer...

/* || Stili Barra Laterale */

Qui inserire gli stili per la barra di navigazione laterale...

/* || Inserti */

Qui inserire gli stili per le componenti inseribili (eg: lo stile per le note, osservazioni, ecc...)

/* || Utility */

Qui inserire gli stili per le classi di utility e simili (eg: .clearfix)...

/* || Contenuto Principale */

Qui inserire lo stile per il contenuto principale...

/* || Stili dark-mode */

Qui inserire le varianti di stile per la dark-mode...

/* || Stili per i Breakpoints */

Qui inserire le varianti di stile ai vari breakpoints...

esistono diversi schemi e logiche di categorizzazione già pronti; nessuno è necessariamente meglio dell'altro, ma eccone uno: SMACS.
