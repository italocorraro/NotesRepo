Appunti sulle media queries in CSS

Questa pagina copre l'argomento della responsività delle pagine web con Media Queries in CSS.

Overview

Le Media Query permettono di applicare regole di CSS selettivamente in base a delle regole sulle caratteristiche della viewport.

Media Query

La sintassi di base per le media queries è

  • il tipo di media si riferisce o a uno schermo (con un browser) (screen) oppure a un dispositivo tipo stampante (per stampare il documento) (print)
  • un'espressione (relativa alla viewport) che deve essere rispettata perché il contenuto della query venga applicato
  • un set di regole CSS da utilizzare se l'espressione è corretta

si possono impostare più regole per la stessa query separandole con un connettore logico and, una virgola o not

Media Rules

Le regole più utili da impostare per le Media Queries:

Altezza e Larghezza

Applicare selettivamente delle regole CSS in base alle dimensioni della viewport

Regola Descrizione
max-height il CSS è applicato solo se la viewport ha altezza minore o uguale al valore specificato
min-height il CSS è applicato solo se la viewport ha altezza maggiore o uguale al valore specificato
height il CSS è applicato solo se la viewport ha altezza uguale al valore specificato
max-width il CSS è applicato solo se la viewport ha larghezza minore o uguale al valore specificato
min-width il CSS è applicato solo se la viewport ha larghezza maggiore o uguale al valore specificato
width il CSS è applicato solo se la viewport ha larghezza uguale al valore specificato
aspect-ratio il CSS è applicato solo se la viewport rapporto larghezza / altezza uguale al valore specificato

le misure della viewport includono anche le scrollbar

Orientamento Schermo

La regola può anche basarsi sull'orientamento schermo rilevato, cioè alle dimensioni relative di altezza e larghezza della viewport

Regola Descrizione
portrait il CSS è applicato solo se la viewport ha altezza maggiore della larghezza (il dispositivo è lungo)
landscape il CSS è applicato solo se la viewport ha larghezza maggiore dell'altezza (il dispositivo è largo)

Impostazioni Utente

Alcune regole possono essere applicate selettivamente in base alle impostazioni del browser

Regola Descrizione
prefers-color-scheme il CSS è applicato solo se è impostata la modalità chiara il CSS è applicato solo se è impostata la modalità scura
prefers-reduced-motion il CSS è applicato se l'utente preferisce movimento ridotto
(nell'esempio vengono disabilitate le animazioni e transizioni di CSS)

Supporto Feature

Alcune regole possono essere applicate in base ad alcune caratteristiche del dispositivo

Regola Descrizione
hover il CSS è applicato solo se il dispositivo supporta l'hover
pointer il CSS è applicato se il dispositivo supporta un cursore ad alta precisione il CSS è applicato se il dispositivo supporta un cursore a bassa precisione (ad esempio un dito su un touchscreen) il CSS è applicato se il dispositivo non possiede una funzionalità di puntamento

Breakpoints

Non ci sono specifiche linee guida da usare per i breakpoint quando si costruiscono design responsivi con media queries, tuttavia, è genericamente consigliato costruire il layout della pagina con un approccio mobile-first, cioè un layout destinato a dispositivi mobili, dallo schermo piccolo e poi procedere sfruttando le queries per aggiungere oggetti a schermo che su un dispositivo mobile avrebbero occupato troppo spazio

Alcune sequenze di breakpoint utili:

Breakpoints Comuni

Breakpoint Descrizione
< 480px Dispositivi Mobili
480px - 768px iPad e Tablet
768px - 1024px Schermi Medi e Laptop
1024px - 1200px Schermi Grandi e Desktop
> 1200px Schermi XL e TV

Breakpoints di Bootstrap

Breakpoint Descrizione
< 576px XS
≥ 576px Small
≥ 768px Medium
≥ 992px Large
≥ 1200px XL
≥ 1400px XXL

Breakpoints di Tailwind

Breakpoint Descrizione
< 640px XS
≥ 640px Small
≥ 768px Medium
≥ 1024px Large
≥ 1280px XL
≥ 1536px XXL

Mobile-First

Un approccio di questo tipo di solito prevede un layout iniziale composto da una singola colonna (solitamente un flex) dove si distribuiscono l'header con una barra di navigazione ridotta (di solito composta di soli pulsanti per aprire i menu del sito), un corpo principale e infine un footer;

da questo layout iniziale si procede poi ad estrarre i menu più importanti per affiancarli alla pagina sugli schermi abbastanza grandi
